开发指南

从零到一，构建你的第一个 MyClaw Skill
最简 Skill

一个 Skill 最少只需要一个 SKILL.md 文件。它使用 YAML frontmatter 声明元信息，markdown 正文作为 Skill 的系统提示词。

my-skill/ SKILL.md
SKILL.md
--- name: my-skill version: 1.0.0 description: 我的第一个 Skill author: your-name --- # 系统提示词 你是一个专业的助手，帮助用户完成任务。
提示：SKILL.md 是每个 Skill 的入口文件，宿主应用会解析 frontmatter 获取元信息，正文部分会被注入为 AI 的系统提示词。
添加可视页面

在 Skill 目录中添加 HTML 文件，即可拥有可视化界面。页面会在 iframe 中加载，通过 postMessage 与宿主通信。

my-skill/ SKILL.md view.html
HTML
<!DOCTYPE html> <html lang="zh-CN"> <head> <meta charset="UTF-8"> <title>My Skill</title> </head> <body> <div id="app">加载中...</div> <script> window.addEventListener("message", (e) => { if (e.data?.type === "skill-data") render(e.data.payload); }); function render(data) { document.getElementById("app") .innerHTML = JSON.stringify(data); } </script> </body> </html>
提示：在 SKILL.md 的 frontmatter 中添加 pages: [view.html] 来声明页面入口。
postMessage 通信

Skill 页面运行在沙箱 iframe 中，通过 postMessage 与宿主应用双向通信。以下是完整的消息类型说明。

接收数据

宿主向 Skill 页面发送以下类型的消息：

JavaScript
window.addEventListener("message", (e) => { const { type, payload } = e.data || {}; switch (type) { case "skill-data": // 初始数据加载，页面首次打开时触发 render(payload); break; case "skill-update": // 增量数据更新，数据变化时触发 render(payload); break; case "skill-progress": // 进度通知，长任务执行期间触发 updateProgress(payload.percent, payload.message); break; } });
发送回调

Skill 页面通过 skill-callback 类型向宿主发送操作请求：

JavaScript
// 通用回调格式 window.parent.postMessage({ type: "skill-callback", action: "navigate", // 操作类型 data: { page: "guide.html" } // 操作参数 }, "*"); // 常用 action 示例： // "navigate" — 页面导航 // "save" — 保存数据 // "refresh" — 请求刷新数据 // "toast" — 显示提示消息
多页面导航

一个 Skill 可以包含多个 HTML 页面。在 SKILL.md 中声明所有页面，通过 navigateTo 函数切换。宿主会处理 iframe 的 src 切换。

JavaScript
// 页面导航函数 function navigateTo(page) { window.parent.postMessage({ type: "skill-callback", action: "navigate", data: { page } }, "*"); }
HTML — 导航栏模板
<nav class="nav"> <button class="nav-btn active" onclick="navigateTo('view.html')"> 仪表盘 </button> <button class="nav-btn" onclick="navigateTo('guide.html')"> 开发指南 </button> </nav>
提示：每个页面需要自行维护导航栏的 active 状态。建议将导航栏样式抽为公共 CSS 片段，通过复制保持一致。
