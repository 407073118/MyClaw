# 分层约束

## 1. 根层依赖规则

- 根层可以引用 `desktop/` 与 `cloud/` 的公开目录与文档入口。
- 根层不能把某个工作区的局部实现细节写成全仓库公共真理。

## 2. 工作区级约束

- `desktop/` 与 `cloud/` 互为平级工作区。
- 平级工作区之间不应直接形成代码级耦合。
- 共享只允许通过明确的契约、文档或发布边界发生，而不是跨目录直接复用内部实现。

## 3. 文档分层规则

- `AGENTS.md`
  - 只放当前层必须立即知道的规则、导航、禁区。
- `docs/architecture/*`
  - 放稳定结构、边界、依赖规则。
- `docs/product/*`
  - 放目标、需求、排期、功能计划。
- `docs/design/*`
  - 放接口、数据、流程设计。
- `docs/processes/*`
  - 放协作与交付流程。
- `docs/agents/*`
  - 放 Agent 阅读顺序、行为边界、角色说明。

## 4. 协议变更规则

- 改共享契约前先确认消费者范围。
- 改契约后必须同步更新调用端、文档和测试。
- 没有同步验证，不应宣称协议变更完成。

## 5. 实施规则

- 根层任务只改根层文件。
- 工作区任务优先改工作区内文件。
- 子项目任务优先改最近层目录，不向上扩散无关改动。
