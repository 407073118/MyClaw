# 业务域与边界

## 1. 根仓库域

根仓库是治理域，不是业务实现域。

负责：

- 统一规范。
- 跨工作区导航。
- 根层流程与 Agent 说明。

不负责：

- 桌面端功能实现。
- 云端业务实现。
- 子项目内部设计细节。

## 2. `desktop/` 域

`desktop/` 负责本地端能力：

- UI 交互
- 本地 runtime
- Skills / MCP
- 员工能力
- 工作流编排

## 3. `cloud/` 域

`cloud/` 负责云端能力：

- 认证
- Hub 目录
- 工件元数据与下载
- 安装留痕

## 4. 文档域边界

- 根 `docs/architecture/*`
  - 描述跨工作区结构与依赖边界。
- 根 `docs/product/*`
  - 描述跨工作区产品目标与计划。
- 根 `docs/design/*`
  - 描述跨工作区设计入口与聚合说明。
- 根 `docs/processes/*`
  - 描述仓库统一流程。
- 根 `docs/agents/*`
  - 描述 Agent 如何读文档、如何协作、哪些事情不能做。

## 5. Agent 工作边界

- 在根层判断任务落点。
- 一旦确定是 `desktop/` 或 `cloud/` 任务，就切换到对应目录最近一层 `AGENTS.md`。
- 不在根层长时间停留做深度实现。
