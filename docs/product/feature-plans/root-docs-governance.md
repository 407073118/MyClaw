# 根层文档治理计划

## 目标

把根仓库文档收敛为标准目录结构，使根层只承担治理与入口职责，不再和工作区局部文档混杂。

## 范围

- 重写根 `AGENTS.md`
- 建立标准 `docs/` 目录骨架
- 把上下文工程、流程、架构说明迁移到正确分类
- 为 Agent 提供明确阅读顺序

## 非目标

- 不替代 `desktop/` 与 `cloud/` 工作区内部文档
- 不改动具体业务代码

## 验收标准

- 根层 `AGENTS.md` 可作为机器入口直接使用
- 根层 `README.md` 可作为人类入口直接使用
- `docs/` 目录符合标准分类
- 根层不再出现职责混杂的文档摆放
