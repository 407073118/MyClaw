# 模型网关迁移说明

# 背景

为了应对日益增长的模型调用业务增长与基础设施可扩展性需求，我们将对模型服务进行统一新网关化的迁移转换。

此次变更不仅涉及模型网关底层架构与资源的升级，还涉及到客户端调用方式的整体切换。

# 变更内容

## 现状描述

先阶段自建模型的调用方式有三种，分别支持的情况如下

|  | legacy网关 | 旧网关 | **新网关（迁移目标）** |
| --- | --- | --- | --- |
| 预发域名 | pre-llm-inference-api.brapp.com | api-pre.cybotforge.100credit.cn | **api-cybotforge-pre.brapp.com** |
| 预发 ip+端口 | 10.150.108.10:80 | 10.150.108.10:80 | **10.150.109.1:80** |
| 预发是否需要鉴权 | 否 | 是 | **是** |
| 预发是否限速 | 否 | 是 | **是** |
| 生产域名 | llm-inference-api.brapp.com<br>或<br>llm-inference.brapp.com<br>（不同集群必须使用不同域名） | api.cybotforge.100credit.cn<br>（不同集群必须使用不同域名） | **api-cybotforge.brapp.com** |
| 生产 ip+端口 | 10.150.100.10:80 | 10.150.100.10:80 | **10.150.100.100:80** |
| 生产是否需要鉴权 | 否 | 是 | **是** |
| 生产是否限流 | 否 | 是 | **是** |
| 是否需要配置hosts解析 | 是 | 是 | **否** |

## 重要事项

以下事项非常重要，请仔细查看

1.  **legacy模式已几乎废弃，使用legacy模式调用模型的业务方请尽快迁移到新网关（直接跳过旧网关）**
    
2.  **旧网关为基础设施过渡状态产物，扩展性受限，建议统一切换到新网关**（新网关支持代理多个GPU集群的模型服务，普通API用户无需考虑服务所在GPU集群信息）
    
3.  **新网关与旧网关使用相同API Key，无需额外申请。**使用时，若出现报错"401 Unauthorized"，表明没传API Key；若出现"403 Forbidden"，表明API鉴权失败；
    
4.  **新网关下，预发模型调用默认开启限速（10 requests in 10 seconds）**使用时，若出现报错"429 Too Many Requests"，表明触发超速；**生产模型调用限速规则依据项目需求而定；**
    
5.  预发API Key暂时以用户维度发放，生产API Key暂时以项目维度发放，不同环境的API Key前缀格式会有不同；
    
6.  **本次迁移仅针对存量模型服务，考虑到业务平滑性，设置迁移双轨并行期：2026-04-14 至 2026-05-15（期间新旧网关并行，均可承接业务流量）。自 2026-05-15 00:00 后，旧网关及底层资源将执行物理下线与路由黑洞处理。请务必在此 deadline 前完成存量模型业务流量的割接与验证。**
    
7.  **本次迁移仅针对存量模型服务，新增模型服务若无特殊需求，必须通过新网关进行调用，不再支持通过legacy方式或旧网关方式调用；**
    

# 调用示例切换

由于业务历史原因，本次变更涉及**存量模型服务的两种迁移路径**：legacy模式 -> 新网关，旧网关 -> 新网关

下面分别举例介绍

## 从legacy切换到新网关模式

curl示例

```shell
# legacy
curl http://llm-inference-api.brapp.com/br-tts-next-custom-v1/v1/audio/speech -H "content-type: application/json" -H 'authorization: Bearer your_key' -X POST -d '{ "model": "br-tts-next-custom-v1", "input": "海绵宝宝我们一起去抓水母吧", "voice":"paidaxing" }'

# 新网关
curl http://api-cybotforge.brapp.com/v1/audio/speech -H "content-type: application/json" -H 'authorization: Bearer your_key' -X POST -d '{ "model": "br-tts-next-custom-v1", "input": "海绵宝宝我们一起去抓水母吧", "voice":"paidaxing"}'

```

python示例

```python

import requests

# legacy网关，需要绑定 hosts (10.150.100.10 llm-inference-api.brapp.com)
old_url = "http://llm-inference-api.brapp.com/br-tts-next-custom-v1/v1/audio/speech"

# 新网关配置：无需 hosts 绑定，直接请求新域名
# 注意：路径已由 /br-tts-next-custom-v1/v1/... 简化为模型无关 /v1/...
new_url = "http://api-cybotforge.brapp.com/v1/audio/speech"


headers = {
    "Content-Type": "application/json",
    "Authorization": "Bearer your_key"
}

payload = {
    "model": "br-tts-next-custom-v1",
    "input": "海绵宝宝我们一起去抓水母吧",
    "voice": "paidaxing"
}

# 发起请求
response = requests.post(new_url, headers=headers, json=payload)
print(f"Status Code: {response.status_code}")

```

## 从旧网关切换到新网关

curl示例

```shell
# 旧网关
curl http://api.cybotforge.100credit.cn/v1/audio/speech -H "content-type: application/json" -H 'authorization: Bearer your_key' -X POST -d '{ "model": "br-tts-next-custom-v1", "input": "海绵宝宝我们一起去抓水母吧", "voice":"paidaxing" }'

# 新网关
curl http://api-cybotforge.brapp.com/v1/audio/speech -H "content-type: application/json" -H 'authorization: Bearer your_key' -X POST -d '{ "model": "br-tts-next-custom-v1", "input": "海绵宝宝我们一起去抓水母吧", "voice":"paidaxing" }'
```

python示例

```python
### 旧网关
import requests

# 旧版配置：需 hosts 绑定
# 注意：路径已由 /br-tts-next-custom-v1/v1/... 简化为模型无关 /v1/...
old_url = "http://api.cybotforge.100credit.cn/v1/audio/speech"

# 新版配置：无需 hosts 绑定，直接请求新域名
# 注意：路径同样为模型无关 /v1/...
new_url = "http://api-cybotforge.brapp.com/v1/audio/speech"

headers = {
    "Content-Type": "application/json",
    "Authorization": "Bearer your_key" 
}

payload = {
    "model": "br-tts-next-custom-v1",
    "input": "海绵宝宝我们一起去抓水母吧",
    "voice": "paidaxing"
}

# 发起请求
try:
    response = requests.post(new_url, headers=headers, json=payload, timeout=30)
    response.raise_for_status()
    print("请求成功，正在处理音频数据...")
except requests.exceptions.HTTPError as e:
    print(f"请求失败，请检查鉴权或网络策略: {e}")
```

# 补充事项

请各调用方协同研发与运维团队，在双规运行的窗口期内完成以下三项调整：

*   **网络策略与路由配置** 新网关已标准化 DNS 解析机制，**调用方无需再绑定 Hosts**。请直接通过新域名发起请求，并确保出网 ACL / 安全组已放行新网关 IP (预发`10.150.109.1` 及生产 `10.150.100.100`) 的 80 端口。
    

*   **代码与配置替换** 排查业务代码、配置文件的 Upstream 路由规则，将旧域名/IP 全量替换为新域名。
    

*   **接口鉴权接入**
    

新网关已强制开启请求鉴权。调用方需确保发起请求时携带合规的鉴权凭证，以避免在切流后遭遇 HTTP 403           阻断。
