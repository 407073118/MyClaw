# Phase 7 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

Scope: `desktop/src/renderer/pages/PersonalPromptPage.tsx` for `/me/prompt`, plus the sidebar top brand area in `desktop/src/renderer/layouts/AppShell.tsx`. No other shell regions or pages are in scope for this revision.

## Surface Contract

- This page is a personal workspace/preferences surface, not a landing page, wizard, or marketing panel.
- Keep the existing dark shell language from `AppShell`: dense, paneled, understated, and aligned to the sidebar/titlebar chrome.
- The sidebar top brand area is shell chrome, not page content. Treat it as the shell identity anchor above navigation, not as a hero card, promo module, or startup splash clone.
- Remove page-local hero composition. Do not use large banner headers, oversized intro blocks, floating promo cards, or decorative radial gradients.
- Use a compact two-pane workstation layout inside the shell content area:
  left pane `65-70%` width for the single prompt editor, right pane `30-35%` width for read-only system context.
- The page should use full shell width with internal padding, not a centered web-style canvas. Page max-width centering is not allowed for this surface.
- Page padding: `24px` on desktop widths, `16px` on narrower widths. Major inter-panel gap: `16px`.
- All page-local surfaces use shared shell tokens only: `var(--bg-base)`, `var(--bg-sidebar)`, `var(--bg-card)`, `var(--glass-border)`, `var(--radius-lg)`, `var(--radius-md)`.
- Corner radius must stay compact. Use `11px` panel corners and `7px` control corners. Do not introduce `20px+` rounded cards on this page.
- Shadow treatment stays restrained and inherited from existing card tokens. No glow-heavy hero shadows.
- Right pane contains exactly three compact inspector panels:
  system summary, detected tags, quick-start examples.
- Save action lives inside the editor panel header/footer, not in a page-level banner action area.
- The shell brand block must visually belong to the same family as `TitleBar`: same glyph language, same neutral-first palette, and the same low-drama desktop posture. Do not introduce a second logo style, glassmorphism showcase, or neon accent treatment in the sidebar top.

## Sidebar Brand Area Contract

- Purpose: communicate product identity and shell orientation before navigation begins. It should feel like a native desktop sidebar header, not a marketing masthead.
- Composition: one full-width compact block inside `sidebar-header`, containing one logo tile and one stacked text group only.
- Copy: show `MyClaw` as the only primary label. Use one muted functional descriptor such as `Desktop Assistant` on the secondary line if a second line is needed. Version/build/channel labels are prohibited in this area.
- Layout: total block height `56px`; internal padding `12px`; gap `12px` between logo tile and text stack; maintain `24px` separation from the first nav item. Keep the header on the existing 8-point spacing rhythm.
- Logo tile: `36px × 36px`, `11px` radius, centered glyph, neutral shell surface with subtle inset light and border. Replace the current bright solid square treatment with a darker paneled tile that sits closer to `var(--bg-card)` than to white.
- Text stack: secondary descriptor `12px / 400 / 1.4`, primary wordmark `14px / 600 / 1.2`. Keep text to two lines maximum and vertically center it against the logo tile.
- Surface treatment: brand block background should use a low-contrast shell panel treatment based on `var(--bg-card)` or `rgba(255,255,255,0.03)`, with `1px` `var(--glass-border)` outline. No hero gradients, spotlight backgrounds, or floating-card theatrics.
- Accent policy: the block is neutral-first. Accent color is not for full-logo fills, version chips, or decorative glows here. If any accent appears, it must be limited to a micro-detail inside the glyph at very low visual area.
- Interaction: non-clickable and non-draggable for this phase. No pointer cursor, hover lift, press animation, pulse, unread dot, or active route treatment. Navigation emphasis starts with the first nav item, not the brand block.
- State policy: do not place runtime status, sync state, or model/version metadata in the brand area. Operational state belongs in the footer user card or page-level status elements.
- Anti-patterns: no version number, no release channel badge, no slogan/tagline sentence, no oversized logo cube, no centered splash composition, no bright white tile against the dark shell.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, status dots, inline separators |
| sm | 8px | Compact control gaps, chip padding |
| md | 16px | Default panel spacing, field spacing |
| lg | 24px | Page padding, panel interior padding |
| xl | 32px | Split-pane breathing room when needed |
| 2xl | 48px | Reserved for shell-level section breaks only |
| 3xl | 64px | Not used inside this page; shell-level maximum only |

Exceptions: inherit existing shell chrome only: `36px` title bar height, `240px` sidebar width, `56px` sidebar brand block height, `36px` sidebar logo tile, and `24px` sidebar prompt shortcut control.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 | 1.6 |
| Label | 12px | 600 | 1.4 |
| Heading | 20px | 600 | 1.3 |
| Display | 24px | 600 | 1.2 |

Rules:
- Use only weights `400` and `600`.
- The page title must use `Heading` or `Display` only once and must not exceed `24px`.
- Support copy is capped at two short lines. No long marketing-style subtitle block.
- Editor text uses `Body`; metadata, chips, and helper text use `Label`.
- Sidebar brand descriptor uses `Label`; sidebar brand wordmark uses `Body` at `600`. Do not introduce a unique oversized logo type scale for the shell header.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #0C0C0C | Shell content background, page backdrop, editor surround |
| Secondary (30%) | #121212 | Sidebar-adjacent surfaces, panels, nav-adjacent cards |

Accent reserved for: save button fill/border, textarea focus state, generated tag chips, dirty or saved status indicators, existing shell active markers, and optionally a tiny brand-glyph detail. ## Interaction Contract

- Single-record model only: one long-term prompt, one editor, one save action. No multi-step form, tabs, accordions, or repeated sections.
- The sidebar top brand area is informational only in this phase. It must not compete with navigation by behaving like a home button, command surface, or workspace switcher.
- The editor is the primary focus target. When no saved prompt exists, place focus in the textarea on entry.
- Header structure is compact:
  page title, one-line explanation, and compact metadata on the same visual plane.
- Remove the separate prominent meta card. Last-updated information belongs in a small status row or header chip, not its own featured card.
- The editor panel contains:
  panel label, title, character count chip, textarea, inline status row, and save action.
- Character count is secondary metadata only. It must not visually compete with the save CTA.
- Save button is disabled when the draft is pristine or while saving.
- `Cmd+S` and `Ctrl+S` must trigger save when the draft is dirty.
- Saving is inline and non-modal. Keep draft content in place and report result in the status row.
- Success feedback is subtle and local to the editor panel.
- Error feedback is local, red, and preserves the draft.
- The summary and tags panels are read-only system outputs and must stay visually subordinate to the editor.
- Example items are compact utility rows, not large marketing cards. Each item shows a short role cue and truncated preview, not a full-page testimonial feel.
- Clicking an example when the draft is dirty must require confirmation. Clicking an example when the draft is pristine may apply immediately.
- Navigating away with unsaved edits must show a confirmation prompt.
- The sidebar brand block must remain visually stable across route changes. Do not animate the logo, fade labels, or swap metadata when navigation changes.
- On widths below `1180px`, collapse to a vertical stack with editor first and inspector panels second. Preserve compact panel spacing and avoid oversized mobile-web card styling.

## Explicit Anti-Patterns

- No hero banner
- No page-local background gradient or spotlight effect
- No 34px+ headline
- No large isolated meta summary card
- No 16:9 promo composition or landing-page rhythm
- No oversized testimonial-style example cards
- No bright accent flood across the page
- No version/build badge in the sidebar brand area
- No clickable pseudo-home brand block for this phase
- No bright white logo tile against the sidebar
